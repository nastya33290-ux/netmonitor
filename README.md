# NetMonitor

Кроссплатформенная Java-программа для мониторинга сетевых соединений и HTTP(S) запросов с прокси, правилами блокировки, отправкой собственных запросов и **запуском приложений через встроенный прокси**.

## Скачать и запустить

Готовый JAR лежит в папке [`dist/netmonitor.jar`](dist/netmonitor.jar).

Нужна **Java 17+**:

```bash
java -jar netmonitor.jar
```

## Возможности (GUI, 7 вкладок)

1. **Соединения** — таблица активных сетевых соединений ОС с привязкой к процессам (PID + имя), фильтр по конкретному приложению, поиск, авто-обновление.
   - Windows: `netstat -ano` + `tasklist`
   - Linux: `ss` / `netstat`
   - macOS: `lsof`
2. **Прокси / Перехват** — встроенный HTTP(S) прокси: видит метод, URL, заголовки, тело, статус/размер ответа. Быстрые действия «Блокировать хост» и «В отправщик».
3. **Запуск через прокси** *(новое в 1.1.0)* — запускает выбранное приложение с трафиком, направленным через встроенный прокси (переменные `HTTP(S)_PROXY` и/или флаги вроде `--proxy-server`). Прокси поднимается автоматически, трафик сразу виден во вкладке «Прокси / Перехват», где его можно смотреть и блокировать. Есть шаблоны (Chrome/Chromium, curl), список запущенных процессов и вывод процесса.
4. **Отправить запрос** — мини-Postman: любой метод, заголовки, тело, опционально через свой прокси.
5. **Правила блокировки** — блокировка по хосту (`*.ads.com`), URL (подстрока/regex), методу, направлению.
6. **Настройки** — порт прокси, интервалы, уровень логов, автозапуск.
7. **Лог / Отладка** — живой лог DEBUG/INFO/WARN/ERROR.

## Консольный режим (без GUI)

```bash
java -jar netmonitor.jar --cli connections      # список соединений
java -jar netmonitor.jar --cli send GET <url>   # отправить запрос
java -jar netmonitor.jar --cli proxy [порт]     # запустить прокси (по умолч. 8888)
java -jar netmonitor.jar --help
```

## Как ловить трафик приложения

Настрой в приложении/ОС HTTP-прокси `127.0.0.1:8888` (или свой порт). Тогда NetMonitor увидит и сможет блокировать запросы.

- Для **HTTP** виден весь запрос/ответ.
- Для **HTTPS** содержимое зашифровано — прокси работает туннелем: видит и блокирует по **хосту**, но не тело (без установки MITM-сертификата).

## Сборка из исходников

```bash
mvn clean package
# результат: target/netmonitor.jar
```

## Автоматический релиз (GitHub Actions)

В репозитории настроены два workflow:
- **Build** — собирает JAR при каждом пуше в `main` (артефакт можно скачать со страницы Actions).
- **Build & Release** — при пуше тега `v*` собирает JAR и автоматически создаёт GitHub Release с прикреплённым файлом.

Чтобы выпустить релиз, создай и запушь тег:

```bash
git tag v1.0.0
git push origin v1.0.0
```

После этого во вкладке **Releases** появится `NetMonitor v1.0.0` с готовым `netmonitor.jar`.

Конфиги хранятся в `~/.netmonitor/` (`settings.properties`, `rules.tsv`).
