# NetMonitor

Кроссплатформенная Java-программа для мониторинга сетевых соединений и HTTP(S) запросов с прокси, правилами блокировки и отправкой собственных запросов.

## Скачать и запустить

Готовый JAR лежит в папке [`dist/netmonitor.jar`](dist/netmonitor.jar).

Нужна **Java 17+**:

```bash
java -jar netmonitor.jar
```

## Возможности (GUI, 6 вкладок)

1. **Соединения** — таблица активных сетевых соединений ОС с привязкой к процессам (PID + имя), фильтр по конкретному приложению, поиск, авто-обновление.
   - Windows: `netstat -ano` + `tasklist`
   - Linux: `ss` / `netstat`
   - macOS: `lsof`
2. **Прокси / Перехват** — встроенный HTTP(S) прокси: видит метод, URL, заголовки, тело, статус/размер ответа. Быстрые действия «Блокировать хост» и «В отправщик».
3. **Отправить запрос** — мини-Postman: любой метод, заголовки, тело, опционально через свой прокси.
4. **Правила блокировки** — блокировка по хосту (`*.ads.com`), URL (подстрока/regex), методу, направлению.
5. **Настройки** — порт прокси, интервалы, уровень логов, автозапуск.
6. **Лог / Отладка** — живой лог DEBUG/INFO/WARN/ERROR.

## Консольный режим (без GUI)

```bash
java -jar netmonitor.jar --cli connections      # список соединений
java -jar netmonitor.jar --cli send GET <url>   # отправить запрос
java -jar netmonitor.jar --cli proxy [порт]     # запустить прокси (по умолч. 8888)
java -jar netmonitor.jar --help
```

## Как ловить трафик приложения

Настрой в приложении/ОС HTTP-прокси `127.0.0.1:8888` (или свой порт). Тогда NetMonitor увидит и сможет блокировать запросы.

- Для **HTTP** виден весь запрос/ответ.
- Для **HTTPS** содержимое зашифровано — прокси работает туннелем: видит и блокирует по **хосту**, но не тело (без установки MITM-сертификата).

## Сборка из исходников

```bash
mvn clean package
# результат: target/netmonitor.jar
```

Конфиги хранятся в `~/.netmonitor/` (`settings.properties`, `rules.tsv`).
