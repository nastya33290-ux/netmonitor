# NetMonitor v1.1.0

Добавлен запуск приложений напрямую через встроенный прокси.

## Скачать
- Готовый исполняемый JAR: [`dist/netmonitor.jar`](dist/netmonitor.jar) (нужна Java 17+)
- Запуск: `java -jar netmonitor.jar`

## Новое в 1.1.0
- **Вкладка «Запуск через прокси»** — запускает выбранное приложение с трафиком, направленным через встроенный прокси:
  - проставляет переменные окружения `HTTP_PROXY` / `HTTPS_PROXY` / `ALL_PROXY` (и в нижнем регистре);
  - поддерживает флаги запуска (шаблон для Chrome/Chromium: `--proxy-server=...`);
  - прокси поднимается автоматически, если ещё не запущен;
  - список запущенных процессов с остановкой и просмотром вывода;
  - кнопка перехода во вкладку «Прокси / Перехват» — там трафик приложения сразу виден, его можно смотреть и блокировать.

## Базовые возможности
- Мониторинг соединений всего ПК с привязкой к процессам, фильтр по приложению (Windows / Linux / macOS).
- HTTP(S) прокси с перехватом: метод, URL, заголовки, тело, статус/размер ответа.
- Блокировка по хосту (glob `*.ads.com`), URL (подстрока/regex), методу и направлению.
- Отправка собственных запросов (мини-Postman), опционально через свой прокси.
- Настройки, панель логов/отладки.
- Консольный режим: `--cli connections | send | proxy`.

## Известные ограничения
- HTTPS перехватывается в режиме туннеля: блокировка и лог по хосту, без расшифровки тела (нужен MITM-сертификат — не входит в релиз).
- Маршрутизация через прокси работает для приложений, уважающих переменные `HTTP(S)_PROXY` или принимающих флаг прокси (большинство CLI-утилит, Chromium-браузеры). Firefox использует прокси из настроек профиля.
- Вкладка соединений использует системные утилиты ОС (`netstat`/`ss`/`lsof`) — они должны быть доступны в системе.
