# NetMonitor v1.2.0

Детект дочерних процессов запущенного приложения и их автоматический перехват.

## Скачать
- Готовый исполняемый JAR: [`dist/netmonitor.jar`](dist/netmonitor.jar) (нужна Java 17+)
- Запуск: `java -jar netmonitor.jar`

## Новое в 1.2.0
- **Детект дочерних процессов**: если запущенное через прокси приложение порождает другой процесс (браузер открывает рендерер/хелпер, лаунчер запускает дочернюю программу и т.д.), NetMonitor обнаруживает его через `ProcessHandle.descendants()`.
- **Автоматический перехват дочерних процессов**: дочерние процессы наследуют окружение родителя (включая `HTTP_PROXY` / `HTTPS_PROXY` / `ALL_PROXY`), поэтому их трафик сразу идёт через встроенный прокси и виден во вкладке «Прокси / Перехват».
- В таблице процессов теперь видны и родитель, и дочерние процессы (колонки **PID / PPID / Тип / Команда / Статус**); любой из них можно остановить.
- В вывод и в лог пишется событие появления нового дочернего процесса.

## Новое в 1.1.0
- Вкладка «Запуск через прокси»: запуск приложения с трафиком через встроенный прокси (env-переменные + флаги вроде `--proxy-server`), авто-подъём прокси, список процессов, вывод.

## Базовые возможности
- Мониторинг соединений всего ПК с привязкой к процессам, фильтр по приложению (Windows / Linux / macOS).
- HTTP(S) прокси с перехватом: метод, URL, заголовки, тело, статус/размер ответа.
- Блокировка по хосту (glob `*.ads.com`), URL (подстрока/regex), методу и направлению.
- Отправка собственных запросов (мини-Postman), опционально через свой прокси.
- Настройки, панель логов/отладки.
- Консольный режим: `--cli connections | send | proxy`.

## Известные ограничения
- HTTPS перехватывается в режиме туннеля: блокировка и лог по хосту, без расшифровки тела (нужен MITM-сертификат — не входит в релиз).
- Перехват дочерних процессов работает, если они наследуют окружение родителя (стандартное поведение) и уважают переменные `HTTP(S)_PROXY` либо запущены с флагом прокси.
- Вкладка соединений использует системные утилиты ОС (`netstat`/`ss`/`lsof`).
