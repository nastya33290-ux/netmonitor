# NetMonitor v1.3.0

Детект сторонних приложений, открытых отслеживаемым (паттерн bootstrapper → launcher), и их перехват.

## Скачать
- Готовый исполняемый JAR: [`dist/netmonitor.jar`](dist/netmonitor.jar) (нужна Java 17+)
- Запуск: `java -jar netmonitor.jar`

## Новое в 1.3.0
- **Детект открытых приложений, а не только дочерних процессов**: вместо обхода дерева потомков теперь сканируются все процессы системы, и новый процесс считается связанным, если в цепочке его предков есть любой уже отслеживаемый PID.
  - Это ловит классический случай, когда одно приложение запускает **другое приложение**, которое затем «отвязывается» от родителя (например, **bootstrapper открывает launcher**, а лаунчер переподвязывается к init/explorer).
  - Однажды обнаруженный процесс остаётся отслеживаемым — связь не теряется даже после завершения промежуточного (bootstrapper) процесса. Его собственные запуски тоже отслеживаются транзитивно.
- В таблице процессов колонка **Тип** теперь различает «запущено нами» и «открыто приложением» (PID / PPID / Тип / Команда / Статус).
- Событие обнаружения пишется в вывод и лог.
- **Перехват** таких приложений работает автоматически: окружение (включая `HTTP_PROXY` / `HTTPS_PROXY` / `ALL_PROXY`) копируется при exec и наследуется по всей цепочке запусков независимо от переподвязки в дереве процессов.

## Предыдущие версии
- **1.2.0** — детект дочерних процессов через дерево потомков.
- **1.1.0** — вкладка «Запуск через прокси»: запуск приложения с трафиком через встроенный прокси.

## Базовые возможности
- Мониторинг соединений всего ПК с привязкой к процессам, фильтр по приложению (Windows / Linux / macOS).
- HTTP(S) прокси с перехватом: метод, URL, заголовки, тело, статус/размер ответа.
- Блокировка по хосту (glob `*.ads.com`), URL (подстрока/regex), методу и направлению.
- Отправка собственных запросов (мини-Postman), опционально через свой прокси.
- Настройки, панель логов/отладки. Консольный режим: `--cli connections | send | proxy`.

## Известные ограничения
- HTTPS перехватывается в режиме туннеля: блокировка и лог по хосту, без расшифровки тела (нужен MITM-сертификат — не входит в релиз).
- Перехват открытого приложения работает, если оно наследует окружение цепочки запусков (стандартное поведение) и уважает `HTTP(S)_PROXY` либо запущено с флагом прокси. Если приложение запущено системной службой с чистым окружением — перехват через наследование не сработает.
- Детект опирается на сканирование процессов раз в секунду; крайне короткоживущие промежуточные процессы теоретически могут быть пропущены.
- Вкладка соединений использует системные утилиты ОС (`netstat`/`ss`/`lsof`).
